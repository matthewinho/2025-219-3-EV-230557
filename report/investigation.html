Issue Encountered in My Project While working on my project, I encountered an issue when iterating through a column containing the number of streams for each song. This column was expected to contain only integers, but I found a string entry that caused errors in my code. Since numerical operations like summing or comparing values require consistent data types, this inconsistency disrupted my analysis. Such issues often arise in datasets due to data entry errors or inconsistencies from multiple sources. To resolve this, I used Python's try-except method to check each entry—attempting to convert values to integers and skipping any invalid ones. This ensured that only numerical values remained, allowing my analysis to proceed without errors. This experience highlighted the importance of data validation and cleaning in ensuring reliable analysis. Proper validation helps detect inconsistencies early, making datasets more accurate and trustworthy.

Algorithm for Cleaning Invalid Data in a Dataset: 1,Initialize the dataset (the list of song streams) which may contain both valid integers and invalid string values. 2,Create an empty list or new dataset to store valid data. 3,For each item in the dataset (iterate through each entry): a. Check if the item is an integer: i. If the item can be converted to an integer, append it to the new list of valid data. ii. If the item cannot be converted to an integer (it’s a string or invalid), skip it or remove it. 4,Return the cleaned dataset (list with only valid integers). 5,Perform necessary operations (such as summing, sorting, etc.) on the cleaned dataset. End.

Pseudocode for Python (Calculating Average BPM from a List)

SET bpm_list to a list of BPM values (e.g. [120, 130, 125, 140, 115])
SET total_bpm to 0
FOR each bpm in bpm_list ADD bpm to total_bpm END FOR
SET average_bpm to total_bpm divided by the number of elements in bpm_list
PRINT "The average BPM is: ", average_bpm
END
